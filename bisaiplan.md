# 计算机设计大赛“算法设计与应用”赛道规划书 (修正版)

> **目标**: 冲击一等奖
> **项目核心**: 基于现有智能手表数据，利用 **大模型 (LLM)** 增强传统算法（岐黄、扁鹊、嘉庆）的精准度与可解释性。

## 一、 现状评估与核心竞争力

### 1. 核心创新点 (The "First Prize" Factor)
传统的健康监测系统大多止步于“数值展示”或“简单阈值告警”。本项目的核心竞争力在于 **“传统算法 + 生成式AI”的融合架构**：
*   **传统算法** (数值计算快): 负责基础特征提取、高频数据流处理。
*   **大模型** (理解能力强): 负责**上下文理解**（判断为何缺失）、**可解释性**（解释为何高危）、**误报过滤**（结合场景判断跌倒）。

这种**“快思考(传统) + 慢思考(LLM)”**的架构不仅技术先进，而且能切实解决“误报率高”、“缺乏个性化建议”的行业痛点。

### 2. 现状与差距
*   **基础设施 (优)**: RuoYi-Vue3 框架成熟，SpringBoot 后端与 Vue3 前端已就绪。
*   **数据通路 (优)**: 手表数据采集（心率、血压、位置等）路径已打通。
*   **应用层 (优)**: 已有清晰的“三算法”逻辑框架 (`shuoming.md`)。
*   **算法实现 (缺)**: `python-algorithms` 目录下的具体实现为空，Java 端的 LLM 调用和融合逻辑尚未开发。

## 二、 冲击一等奖规划 (Game Plan)

### 核心亮点打造 (Core Features)

#### 1. 岐黄预处理增强 (Qihuang - Data Quality)
> *利用 LLM 的逻辑推理能力，解决“僵硬”的数据填补问题。*
*   **传统痛点**: 简单均值/插值填充忽略了用户行为（如“睡觉摘表”导致的缺失是正常的，不该填）。
*   **创新实现**:
    *   **输入**: 缺失时段上下文（时间、前序心率、是否充电中、位置）。
    *   **LLM 任务**: 判断缺失原因（摘表/死机/接触不良） -> 智能选择策略（不填/填均值/填趋势）。
    *   **展示**: **数据质量透视**图表，展示 AI 修复的记录数和原由。

#### 2. 扁鹊预测增强 (Bianque - Risk Prediction)
> *利用 LLM 的知识库能力，解决“黑盒”预测无法解释的问题。*
*   **传统痛点**: 模型输出“高风险”，但用户一头雾水，不知道怎么改。
*   **创新实现**:
    *   **RAG (检索增强)**: 结合《中国高血压防治指南》等医学文献。
    *   **LLM 任务**: 解析传统模型（如 XGBoost）的特征重要性 -> 结合用户画像 -> 生成**“私人医生级”健康报告**。
    *   **展示**: 动态生成的**健康建议书**，不仅有分，还有详细的饮食/运动指导。

#### 3. 嘉庆算法增强 (Jiaqing - Anomaly Detection)
> *利用 LLM 的常识推理能力，解决跌倒检测“误报率高”的顽疾。*
*   **传统痛点**: 剧烈运动、拍手常被误判为跌倒。
*   **创新实现**:
    *   **多模态融合**: 融合加速度波形描述 + 时间（凌晨vs下午）+ 地点（卫生间vs公园）。
    *   **LLM 任务**: “侦探模式”推理 —— "下午3点在公园且心率持续高位，大概率是运动并非跌倒"。
    *   **展示**: **误报拦截日志**，演示系统如何成功拦截一次假警报。

## 三、 实施路线图 (Roadmap)

### 第一阶段：算法内核构建 (Algorithm Core) - [3天]
1.  **Python环境**: 搭建 `ueit-ai` 服务，配置 LLM 接口（DeepSeek/ChatGLM/OpenAI）。
2.  **Prompt 工程**: 编写针对上述三个场景的高质量 Prompt 模板。
    *   *例: "分析以下传感器数据序列，判断用户当前行为模式..."*
3.  **传统算法桩**: 实现基础的 Random Forest (扁鹊) 和 阈值法 (嘉庆) 作为对照组。

### 第二阶段：通过 Java 串联业务 (Integration) - [3天]
1.  **数据清洗**: 在 Java 端应用“岐黄”逻辑，调用 Python 清洗数据后再入库。
2.  **异步增强**: 这是一个高耗时操作，需引入消息队列或异步线程池，用户上传数据后，后台静默进行 LLM 分析，完成后推送通知。
3.  **数据库设计**: 扩展表结构，存储 LLM 生成的“解释文本”和“置信度”。

### 第三阶段：驾驶舱与演示设计 (Visualization) - [2天]
1.  **健康驾驶舱**: 
    *   **左侧**: 实时数据流（心率波形）。
    *   **中间**: 人体3D模型（高亮风险部位）。
    *   **右侧**: 实时滚动的 AI 分析日志（"AI 正在思考... 检测到异常心率，结合运动数据分析排除了风险"）。
    *   **交互**: 点击任意一条告警，可以看到 LLM 的详细推理过程。

### 第四阶段：申报材料润色 (Documentation)
1.  **PPT 逻辑**: 强调从“传统监测”到“AGI 赋能健康”的跨越。
2.  **演示视频**: 录制一个“误报拦截”的完整流程。
