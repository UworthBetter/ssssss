# 前端展示优化方案分析

**分析时间:** 2026-03-23
**当前页面:** DashboardView.vue (1135行)
**目标:** 更好地体现处理链: 异常 → 事件 → AI解析 → 风险评估 → 处置建议 → 自动执行/留痕

---

## 📊 现状分析

### 当前展示结构

```
DashboardView.vue
├── 底层: 全屏三维地图 (AMap)
├── 顶部: HUD胶囊信标 (5个KPI)
├── 左侧: 全息渐变翼
│   ├── 价值卡片 (4个指标)
│   ├── 人群画像 (年龄性别表)
│   └── 最近异常 (表格)
├── 右侧: 全息渐变翼
│   ├── 实时数据 (KPI饼图)
│   └── 趋势分析 (折线图)
└── 底部: 事件雷达流 (异常事件表)
```

### 现有数据流

```
API调用:
  • getRecentAbnormal() → recentAbnormal (最近异常)
  • getIndexException() → exceptionList (异常列表)
  • getRealTimeData() → realTimeData (实时数据)
  • getAgeSexGroupCount() → ageSexTable (人群统计)

数据展示:
  • 表格展示异常事件
  • 图表展示统计数据
  • 地图展示位置信息
```

### 现有问题

❌ **缺少处理链的可视化**
- 用户看不到异常是如何被处理的
- 看不到AI的分析过程
- 看不到风险评估的依据
- 看不到处置建议的理由
- 看不到自动执行的结果

❌ **信息层级不清**
- 所有异常平铺展示
- 无法区分异常的严重程度
- 无法看到处理进度

❌ **交互不足**
- 点击异常后没有详情展示
- 无法查看完整的处理链路
- 无法看到AI的决策理由

---

## 🛠️ 实现建议

### 第1步: 新增数据结构

```typescript
// 处理链详情
interface ProcessingChain {
  eventId: string
  stages: [
    {
      name: '异常检测'
      status: 'completed' | 'processing' | 'pending'
      timestamp: string
      details: {
        anomalies: Array<{ type: string; value: string; severity: string }>
      }
    },
    {
      name: '事件产生'
      status: 'completed'
      timestamp: string
      details: { eventId: string; eventType: string }
    },
    // ... 其他阶段
  ]
  totalDuration: number // ms
}
```

### 第2步: 新增API调用

```typescript
// 获取处理链详情
const getProcessingChain = (eventId: string) => {
  return api.get(`/api/ai/event-processing/chain/${eventId}`)
}
```

### 第3步: 新增UI组件

```
新增组件:
  • ProcessingChainPanel.vue - 处理链详情面板
  • ProcessingChainTimeline.vue - 处理链时间轴
  • ProcessingChainProgress.vue - 处理链进度条
  • AnomalyDetailModal.vue - 异常详情弹窗
```

### 第4步: 集成到DashboardView

```typescript
// 在异常列表中添加点击事件
const handleRowClick = (row: ExceptionRow) => {
  // 获取处理链详情
  const chain = await getProcessingChain(row.id)
  // 显示详情面板
  showProcessingChainPanel(chain)
}
```
